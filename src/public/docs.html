API Documentation

Unified internal utility microservice — URL checker + image conversion & compression. Built for n8n automation workflows.

⚡ Fastify 🔷 TypeScript 🖼️ Sharp 🐳 Docker Ready

🖼️ Supported Formats

📥 Input (Upload)

JPEG PNG WebP GIF TIFF AVIF

📤 Output (Result)

WebP JPEG PNG

📡 Endpoints

GET /health Health check ▶

curl

curl http://localhost:3000/health

Response

{
  "status": "ok",
  "uptime": 123.45,
  "timestamp": "2026-03-03T08:00:00.000Z"
}
GET /check?url={url} Check single URL ▶

Query Parameters

ParamTypeRequiredDescription
url string ✅ Full URL to check (include https://)

curl

curl "http://localhost:3000/check?url=https://google.com"

Response

{
  "url": "https://google.com",
  "status": 200,
  "ok": true,
  "redirect": true,
  "finalUrl": "https://www.google.com/",
  "responseTime": 356
}
POST /check Check multiple URLs (max 100) ▶

Request Body — JSON

{
  "urls": [
    "https://google.com",
    "https://github.com"
  ]
}

curl

curl -X POST http://localhost:3000/check \
  -H "Content-Type: application/json" \
  -d '{"urls":["https://google.com","https://github.com"]}'

Response

[
  {
    "url": "https://google.com",
    "status": 200,
    "ok": true,
    "redirect": true,
    "finalUrl": "https://www.google.com/",
    "responseTime": 280
  },
  {
    "url": "https://github.com",
    "status": 200,
    "ok": true,
    "redirect": false,
    "finalUrl": "https://github.com",
    "responseTime": 412
  }
]
💡 Maximum 100 URLs per request. All URLs are validated before processing.
POST /convert Convert image format ▶
📎 Request must be multipart/form-data with the file field named file.

Query Parameters

ParamTypeDefaultDescription
format string webp Output format: webp, jpg, png
quality int 80 Quality 1–100
width int — Resize width in pixels
height int — Resize height in pixels
fit string cover cover, contain, inside, fill, outside

curl — Basic Convert

curl -X POST "http://localhost:3000/convert?format=webp&quality=80" \
  -F "file=@input.png" \
  -o output.webp

curl — Convert + Resize

curl -X POST "http://localhost:3000/convert?format=png&width=400&height=300&fit=cover" \
  -F "file=@photo.jpg" \
  -o resized.png
💡 Response is the binary image with appropriate Content-Type header. Use -o filename with curl to save.
POST /compress Compress image ▶
📎 Request must be multipart/form-data with the file field named file.

Query Parameters

ParamTypeDefaultDescription
quality int 75 Compression quality 1–100
lossless bool false Lossless compression (WebP/PNG)
effort int 4 Compression effort 0–6 (higher = slower but smaller)
format string — Force output format (default: keep original)

curl — Compress JPEG

curl -X POST "http://localhost:3000/compress?quality=60" \
  -F "file=@photo.jpg" \
  -o compressed.jpg

curl — Lossless WebP

curl -X POST "http://localhost:3000/compress?format=webp&lossless=true&effort=6" \
  -F "file=@input.png" \
  -o output.webp

🔗 n8n Integration

Use the HTTP Request node in n8n to connect to internal-tool.

1

URL Check

Check a URL from your workflow data

Method: GET
URL: http://internal-tool:3000/check?url={{ $json.url }}
2

Convert Image

Convert a binary image to WebP

Method: POST
URL: http://internal-tool:3000/convert?format=webp&quality=80
Body Type: Form-Data/Multipart
Parameter: file
Input Field: data
3

Compress Image

Compress a binary image with quality control

Method: POST
URL: http://internal-tool:3000/compress?quality=60
Body Type: Form-Data/Multipart
Parameter: file
Input Field: data
💡 When running both n8n and internal-tool in Docker, use the container name internal-tool as hostname. Add both services to the same Docker network.

⚠️ Error Responses

All errors return JSON with a consistent structure:

{
  "error": "ValidationError",
  "message": "Missing 'url' query parameter",
  "statusCode": 400,
  "requestId": "a1b2c3d4-e5f6-7890"
}
StatusMeaningCommon Cause
400 Bad Request Invalid parameters, missing file, or bad URL format
413 Payload Too Large Uploaded file exceeds MAX_FILE_SIZE (default 10MB)
415 Unsupported Format Image format not supported by Sharp
500 Internal Error Unexpected server error
